# Security Policy

## Supported versions

Security fixes are applied to the latest release.

| Version | Supported |
| --- | --- |
| Latest 1.x release | Yes |
| Earlier releases | No |

## Reporting a vulnerability

Do not open a public GitHub Issue for a suspected vulnerability and do not
include SMTP usernames, passwords, message bodies, or private mail headers in a
report.

Use GitHub's private vulnerability reporting form:

<https://github.com/acodebeard/viazen-mailersend-smtp/security/advisories/new>

Include the affected version, reproduction steps, impact, and suggested
mitigation when available.

Problems with a MailerSend account or the MailerSend service should be reported
to MailerSend, Inc., not through this project's vulnerability form.
